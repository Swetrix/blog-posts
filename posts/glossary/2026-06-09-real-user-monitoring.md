Real user monitoring, often shortened to RUM, measures website performance from actual visitor sessions. Instead of only testing a page in a lab environment, RUM collects performance data from real devices, browsers, countries, and network conditions.

RUM helps teams understand how fast a website feels to real users.

## What RUM measures

Real user monitoring can track:

- Time to First Byte
- First Contentful Paint
- Largest Contentful Paint
- Interaction to Next Paint
- Cumulative Layout Shift
- Page load timing
- Resource timing
- Device and browser context
- Country or region

These metrics can be segmented by page, device, country, browser, and traffic source.

## RUM vs synthetic monitoring

Synthetic monitoring runs controlled tests from selected locations and devices. RUM collects measurements from real visitors.

Synthetic tests are good for consistency and proactive checks. RUM is better for understanding real-world user experience.

## Why RUM matters for analytics

Performance affects bounce rate, conversion rate, SEO, and user trust. RUM helps teams see whether slow pages are hurting real outcomes.
