Error tracking is the practice of collecting, grouping, and analyzing software errors so teams can find and fix problems. For websites and web apps, it often focuses on JavaScript errors that happen in visitors' browsers.

Error tracking matters because analytics can show that conversions dropped, but errors can explain why.

## What error tracking captures

Error tracking tools may capture:

- Error message
- Stack trace
- Page URL
- Browser
- Operating system
- Device type
- Release or version
- Frequency
- First and last occurrence
- Affected users or sessions

This context helps developers reproduce and prioritize fixes.

## Why error tracking belongs near analytics

Conversion drops, funnel drop-offs, and high bounce rates can be caused by technical issues. A broken signup button, failed checkout script, or browser-specific bug can look like a marketing problem in a normal analytics dashboard.

When error tracking is connected to analytics, teams can see whether errors affect important pages, traffic sources, or conversion flows.

## Error tracking and privacy

Error tracking should avoid collecting sensitive user data unnecessarily. Teams should be careful with request payloads, URLs, form values, and session context.
