Personal data is information that identifies a person or can reasonably be linked to a person. Under privacy laws such as GDPR, personal data can include obvious identifiers such as names and email addresses, but it can also include technical and behavioral data in the right context.

In website analytics, personal data matters because analytics systems often collect device, network, location, event, and usage information.

## Personal data examples

Examples can include:

- Name
- Email address
- Phone number
- Account ID
- User ID
- IP address
- Precise location
- Device identifiers
- Cookie IDs
- Payment identifiers
- Behavioral profiles

Whether a data point is personal data can depend on context. A single anonymous page view may not identify someone, but combining it with account data, unique identifiers, or detailed behavior can change the privacy analysis.

## Personal data in analytics

Analytics tools may process personal data when they store IP addresses, persistent identifiers, user-level profiles, precise geolocation, session recordings, or event data tied to accounts.

Privacy-friendly analytics reduces risk by collecting less, avoiding unnecessary identifiers, aggregating reports, limiting retention, and avoiding cross-site tracking.

## Data minimization

Data minimization means collecting only what you need for a specific purpose. For many websites, useful analytics does not require third-party cookies, precise identity, or advertising profiles.
